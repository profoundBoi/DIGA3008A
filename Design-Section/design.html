Part 01

1. IxD documentation process:

1.1. Goal alignment:

My goal is to create a website that is easy to use and navigate. I want to make sure that the website is accessible to all users, including those who may not be familiar with web design or technology. I want to create a website that is visually appealing and easy to read.

My goal was to make a website that is easy for people to use especially people experiencing the web for the first time. I'll do this by displaying information about myself on the website that people can look through. I decided to put my navigations at the top of the page similarly to the way Dropbox service (Dropbox.com, 2025) does theirs. This is easy to see for new users especially if I keep the design consistent on all the pages. When styling I will choose a colour that stands out to the users so that they will know this is the navigation bar. I will be using an unordered tag for my navigation system. My visual hierarchy will prioritise my navigation bar.

1.2. Information structure:

My websites content will be structure as follows: Homepage: This page will have an avatar image of myself with a short about me text along side it. I will also include an index section on this page to help direct users to specific information.

Portfolio: The portfolio page will have all my games for people to play. Entering a game page will take users to an about section of this game and provide a link to the game's website on itch.

Blogs: The blog pages will show all the discussion board assignments in the form of blogs. I want the viewers to see how I went about designing the website and thinking about how media is interactive through my blogs. There will be more navigation options that will direct the user to the blog week that they clicked on. This will also act as my development log.

Inspiration: The inspiration page will show picture and links of all the inspiration that I used for this assignment. I was thinking of adding a mood board section for my style guide in on this page.

Images: the images will just show all the planning drawing and diagrams that I made for the website.

1.2.1. Information structure:

The information structure of my website is going to be a flat structure. This is because I want the user to be able to navigate between the different pages easily without having to go through multiple pages.

    1. Style Guides:

    1.1. Goal and aesthetics:

    This website will have a green shade in terms of colour for my aesthetic. Then the text that we are using is going to have a digital semi-robotic feel to it. The reason I am doing this is because I am a game designer, so I want my website to reflect who I am through its font and colour schemes. If you have a look at my earlier design choices for my website colour schemes you will see that the colours are different to what I have decided to do now. I decided to use green shades because my avatar that is being displayed is green with a robotic feel. If you have a look at my itch page you will find a similar feel. I am going to be using my itch.io page as reference in terms of aesthetics. Originally, I wanted my website to have very minimal details so that the user is not over stimulated when looking at my website. I now found myself adding a little bit more to my aesthetic because I found that the aesthetic I had was underwhelming when compared to the tech game feel that I was trying to create.

    1.2. Visual design elements:

      Navigation -

    • My original design for navigation in my wireframe took up the entire nav section at the top of my website. While styling my navigation bar I realised that this design did not compliment my aesthetic choices in terms of colours and fonts. It did not have a tech game feel to it hence I decided to change it to something more suitable. The new navigation design pushes all my nav links to the far right and stacks them next to each other from there. I want to try and keep my navigation as close to the right-hand margin as possible. Additionally, I added a dynamic background to the navigation text to help the user identify which section of my website they are on.

    • At the start of my html section of this website I designed my blogs to be navigated by using a navigation bar. This was not suitable for that way I wanted my user to switch between my blogs. Using a nav element prevented me from styling my blog navigation as a card. There might be a way to do it by using navigation however I haven't figured it out. As an alternative I decided to just add the href link to a class called blog-card. This class would store only that specific week's information and link. This allowed me to create the card style that I wanted to have. I also positioned the cards in rows so that the blog weeks flow into the next one. I added a background image to the blog cards when the user hovers over it. This communicated to the user which card week they are currently exploring. A read more text which is my actual link to the different blogs has been added to the bottom of each card. Its at the bottom of each card so that if the user decides that they would like to explore my discussions for the topic on the blog card. The read more text also changes colour once the user hovers over it so that they know they can click it.
      Text shadows -

    • I added text shadows to the text that where directly above the background image that I put it. This was done because the text above my background images where hard to read without some sort of contrast between them. It took me a while to get the text shadow to look the way I wanted it to look. It made some of my text look blurry which was hard to read so I add a hard line surrounding the text.
      Containers and container backgrounds -

    • To establish content hierarchy and improve visual clarity, I enclosed major content sections within containers that use semi-transparent dark green backgrounds. These transparent overlays allow the user to see the background image while still clearly distinguishing between different content areas. Initially, I experimented with lighter green tones for these containers, but they lacked contrast with my white text and made the content hard to read. The darker tone not only complements the rest of the palette but also ensures accessibility and readability.
      Web Icon -

    • A favicon has been added to every page of the site. While small, this detail enhances the sense of professionalism and reinforces brand identity. It also makes it easier for users to locate the site tab when multiple browser tabs are open.
      Colour usage -

    • I tried to use a different colour for my nav and my background at first. My nave started off using a green shade that was lighter than the background colour for my body. I then changed the body background to an actual image which was lighter than my nav bar. I found that this brought more life to the website, and it was more inviting for people. The colour palette has changes slightly. I used the colour swatches shown in 1.3 as a basis to find the actual shades of green that best suited my website. Initially I thought that my colour watches in 1.3 were going to be my final selection but as I designed my website and saw how these colours looked in my website I decided to alter them to best suit the aesthetic that I was going for.

    1.3. Colour Swatches:

      Primary - Dark pastel green

I found that this added contrast and appeared to be visually more dominant for my navigation bars and my footers. It had more weight to the website as seen in typical game dashboards and home pages, their navigation is often heavy depending on the style of game being played and designed.

    1.4. Font samples:

      Tektur font from family sans-serif.

      For my headings, I chose the “Tektur” font. This style has a digital, angular appearance that immediately create a feel of sci-fi or tech-inspired aesthetic, which aligns closely with my website's focus on game design and development. The sharp geometry of the font adds personality and makes the headers visually distinct from the rest of the content. It helps to establish a visual hierarchy, drawing attention to key titles and section names.

      For my body text I used the “Open Sans” font. This font is known for its legibility and clean lines, making it an excellent choice for seperating information from the headings.
